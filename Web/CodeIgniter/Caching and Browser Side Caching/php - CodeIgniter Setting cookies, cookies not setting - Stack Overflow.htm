Ok, I figured out my problem. Maybe this will help someone else down the road.

In your CI config.php there are specific settings for cookies/sessions one of those settings is a prefix for cookies. Apparently if that setting is not set eg ='' then somewhere down the line the cookie breaks silently. However if you have it set and more so if you forgot you set it when you configured CI initially when you set a cookie.

$this-input->set_cookie('myCookie');

and you had a prefix of 'fox' for example..

and you went to go looking for your cookie

$this->input->cookie('myCookie');

you will get the equivlant of bool(false) however if you go looking for your cookie like

$this->input->cookie('foxmyCookie');

you will actually get the cookie.

FYI, you can not set cookies for domain 'example.com' from within a domain (HTTP_HOST) with a different name like 'yoursite.com'. it is security issue, browsers will ignore it. check with mozilla firebug 'NET' tab the response headers, if the cookie exists, and not set , then i am right.

secondly, after you set cookie, it will be available on the server (php) , only in next request, since current $_COOKIE still not updated. if CI implemented this differently, this is wrong implementation of the concept.
